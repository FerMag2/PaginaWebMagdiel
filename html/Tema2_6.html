FRONTEND VS BACKEND

El frontend se encarga de la interfaz y la experiencia del usuario en una aplicación web, mientras que el backend se encarga de la lógica, el procesamiento y la gestión de datos en el servidor.

Concepto de Api (mediante boxeo)

Frontend (Boxeador en el Ring): Imaginemos que el frontend es como un boxeador en el ring. Este boxeador es la cara visible de la pelea, lo que el público ve. Él lanza golpes y recibe golpes, representando la interfaz de usuario de una aplicación.

Backend (Entrenador en la Esquina): El backend sería como el entrenador que está en la esquina del ring. Este entrenador tiene un conocimiento profundo y estrategias para la pelea. Él proporciona orientación y lógica para que el boxeador (frontend) pueda pelear eficazmente.

Comunicación mediante apis

API (Instrucciones entre Rounds): En el boxeo, entre rounds, el entrenador puede dar instrucciones al boxeador, proporcionando estrategias específicas para el próximo round. Esto sería similar a una API en desarrollo web. Es como el conjunto de instrucciones que el frontend recibe del backend para solicitar información o realizar acciones específicas.
